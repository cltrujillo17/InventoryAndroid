# CS 360 Module Eight Journal

### Summary of Requirements and Goals  
The app I developed was an **Inventory Management App** designed to help users track warehouse stock. Its primary goal was to allow users to log in securely, view a grid of items, add or remove inventory, adjust quantities, and receive notifications when items reached zero. This design addressed the user need for a simple and reliable way to monitor and maintain inventory levels.  

### Screens and Features  
The app included a **login screen** for user authentication, an **inventory dashboard** showing items in a grid layout, and an **item management form** for adding or updating items. Notifications were also implemented to alert users when stock ran out. My UI design followed Android Material Design guidelines by prioritizing clarity, consistency, and accessibility. These designs were successful because they minimized clutter and gave users straightforward ways to interact with the app.  

### Coding Approach  
When coding, I broke down requirements into smaller pieces and connected the UI elements to backend logic step by step. I used modular methods for login authentication, database interaction, and notification triggers. This approach can be applied to future projects since it promotes maintainability and makes debugging easier.  

### Testing Process  
I tested the app using the Android Emulator by simulating user logins, adding and removing items, and checking that notifications triggered correctly. I also set breakpoints in the code to track database updates. Testing ensured the app was functional and revealed small issues early, like handling empty fields in item forms. This process was important because it verified reliability before considering deployment.  

### Overcoming Challenges  
One challenge I faced was managing the balance between **simplicity and functionality**. At first, I included too many optional features, which risked overwhelming users. I innovated by refactoring the design to keep the app lightweight, focusing only on the most essential functions while leaving room for future enhancements.  

### Areas of Success  
A specific component where I was successful was the **notification system**. I was able to implement real-time alerts when items reached zero, demonstrating my ability to connect system services (like notifications) with database changes. This feature showed both my technical skills and my understanding of user-centered design principles.
